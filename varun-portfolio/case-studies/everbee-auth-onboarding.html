EverBee Auth Setup & Product Selector

A streamlined sign-in and onboarding flow that helps users reach value faster.

RoleProduct Designer
Timeline6 weeks
PlatformWeb App
ToolsFigma · FullStory

New users were signing up but stalling before they ever reached EverBee's core research tools. The signup form, product selector, and first-run setup had all been built at different times by different teams, so the experience felt like three separate products stitched together.

This project unified sign-in and first-run setup into one coherent flow, with the single goal of getting a new user to their first useful insight as fast as possible.

The problem

Funnel data showed roughly a third of new signups never completed onboarding. Support tickets echoed the same confusion: people didn't know which product tier they'd signed up for, or what to do once they landed inside the app.

The product selector — meant to route users to the right tools — was actually adding a decision users weren't ready to make yet.

The approach

01

Audit every drop-off point

Walked the full funnel from landing page to first insight, logging every screen where users could stall or bounce.

02

Defer the hard choice

Moved product selection from a mandatory first step to a contextual suggestion once the user showed intent.

03

Redesign sign-in for speed

Added social auth and cut the credential form from five fields to two, validated live rather than on submit.

04

Build a guided first run

Replaced a static welcome tour with a checklist tied to real product milestones, so progress felt earned, not performative.

The solution

Sign-in now takes under 15 seconds for returning users and offers Google auth up front for new ones. Product selection was removed as a gate entirely — the app recommends a starting point based on the first action a user takes, and lets them switch anytime from settings.

A three-item setup checklist replaced the old onboarding tour, each item linked directly to the real feature rather than a screenshot of it.

The best product selector is the one the user never has to think about.
